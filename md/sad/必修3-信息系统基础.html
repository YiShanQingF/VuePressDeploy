# 1-1 信息系统基础知识（一）

# 信息系统概述

信息系统有计算机硬件、网络和通信设备、计算机软件、信息资源、信息用户以及规章制度组成的处理信息流为目的的人机一体化系统。

信息系统的功能：输入、存储、处理、输出、控制

# 信息系统的发展

诺兰模型 发展阶段：初始阶段、传播阶段、控制阶段、集成阶段、数据管理阶段、成熟阶段

# 信息系统的分类

信息系统类型：业务处理系统TPS、管理信息系统MIS、决策支持系统DSS、专家系统ES、办公自动化系统OAS、

# 信息系统的生命周期

产生：概念、需求分析

开发：总体规划、系统分析、系统设计、系统实施、验收

运行：维护

消亡：归档存储、切换过度

# 信息系统的建设原则

高层人员介入

用户参与开发

自顶向下规划

发展性：超前性

创新：先进性

整体性：完整性

经济性：使用性

工程化

# 信息系统的开发方法

结构化

原型

面向对象

面向服务

# 企业资源规划ERP

企业的所以资源包括三大流：物流、资金流和信息流。ERP也就是对这3种资源进行全面集成管理的系统。

核心模块：

  • 生产预测
  • 销售管理
  • 经营计划
  • 主生产计划
  • 物料需求计划
  • 能力需求计划
  • 车间作业计划
  • 采购与库存管理
  • 财务管理

ERP的功能：

  • 支持决策
  • 为处于不同行业的企业提供有针对性的IT解决方案
  • 从企业内部的供应链发展为全行业和跨行业的供应链

# 1-7 密钥管理技术

秘钥分配要解决的两个问题：

  • 引进自动分配秘钥机制，提高系统效率
  • 尽可能减少系统驻留的秘钥量

对称秘钥分配：

  • 物理手段
  • 由第3方选取秘钥，再通过物理手段分别发送A、B
  • A、B事先有一个秘钥，其中一方选取新秘钥后，用已有的秘钥加密该新秘钥发送
  • 三方A、B、C各有一保密信道，C选取秘钥后，分别通过A、B各自的保密信道发送

# 1-8 访问控制与数字签名技术

主体请求客体过程的访问控制

主体：用户、进程、应用、设备

客体：文件、进程、数据库、设备

类型：

  • 强制访问控制
  • 自主范文控制
  • 基于角色的访问控制

特点：

  • 数字签名用于身份验证
  • 私钥加密、公钥解密

签名方

  • 对明文进行哈希计算产生数字摘要
  • 对数字摘要用私钥进行加密产生数字签名
  • 再将数字签名和明文传输给验证方

验证方

  • 对数字签名使用公钥解密得到数字摘要
  • 对明文进行哈希计算得到数字摘要
  • 两个数字摘要进行匹配

# 1-9 信息安全的抗攻/击技术

  • 拒绝服务攻击DoS，DoS是使系统不可访问并因此拒绝合法的用户请求，侵犯系统的可用性。
  • 分布式拒绝服务攻击DDoS
  • 同步包风爆，是用于最广泛的一种DoS攻击方式，攻击TCP协议建立连接的三次握手，让目标主机等待连接完成而耗尽资源。可以减少等待超时时间来防范。
  • ARP欺骗： ARP协议是解析IP地址为MAC网卡物理地址，欺骗该机制即可阻断正常的网络访问
  • DNS欺骗：
  • IP欺骗：可以用防火墙防范
  • 端口扫描：
  • ICMP攻击： Internet控制报文协议。它是TCP/IP协议簇的一个子协议，用于在P主机、路由器之间传递控制消息。控制消息是指网络通不通、主机是否可达、路由是否可用等网络本身的消息。攻击者可以利用利用操作系统规定的ICMP数据包的最大尺寸不超过64KB这一规定，达到使TCP/P堆栈崩溃、主机死机的效果。防范方法是打补丁、升级到新版操作系统。
  • SNMP攻击： 简单网络管理协议，常用于管理网络设备，早期的SNMP V1协议缺少认证，可能被攻击者入侵。防范方法是升级SNMP协议到V2以上并设置访问密码。

# 1-10 信息安全的保障体系与评估方法

# 计算机信息系统安全保护等级

  • 第1级：用户自主保护级，隔离用户与数据，使用户具备自主安全保护的能力。
  • 第2级：系统审计保护级，通过登录规程，审计安全相关事件和隔离资源，使用户对自己的行为负责。
  • 第3级：安全标记保护级，安全策略模型，数据标记以及主体对客体强制访问控制的非形式化描述；具有准确地标记输出信息能力；消除通过测试发现的任何错误。
  • 第4级：结构化保护级，系统具有相当的抗渗透能力
  • 第5级：访问验证保护级，系统具有很高的抗渗透能力
